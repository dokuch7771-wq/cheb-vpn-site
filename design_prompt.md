## ПРОМПТ:

Ты эксперт по UI/UX и веб-дизайну. Мне нужно полностью переработать визуальный дизайн страницы-дерева знаний для инженерного воркспейса VPN-сервиса.

**Текущая страница приложена** (knowledge_tree.html). Она уже содержит правильную структуру и контент — менять смысловое содержание, иерархию и тексты НЕ нужно. Нужен новый визуальный дизайн.

### Концепция которую хочу

Стиль: **тёмная тема, инженерный/tech эстетика**, вдохновлённая:
- Linear.app (чистота, контраст, типографика)
- Vercel Dashboard (монохромность + акценты)
- Obsidian Canvas (ноды, связи, graphy-feel)

### Требования к дизайну

**Структура и навигация:**
- Дерево должно быть **интерактивным** — узлы можно сворачивать/разворачивать кликом
- При ховере на карточку — подсветка + небольшой shadow
- Уровни должны быть визуально чётко разделены (не просто цветом, а пространством и размером)
- Добавить боковую панель (sidebar) с быстрой навигацией по уровням

**Визуальный стиль:**
- Фон: #090c10 (почти чёрный, чуть синеватый)
- Карточки: стекломорфизм (backdrop-filter: blur) с тонкой границей
- Соединительные линии между уровнями: анимированные (тихий pulse или gradient flow)
- Иконки: использовать lucide-icons или heroicons (CDN)
- Шрифт: Inter для текста, JetBrains Mono для кода

**Карточки по уровням:**
- L1 (Критическое): крупные карточки, красная акцент-полоса, badge "CRITICAL"
- L2 (Важное): средние карточки, оранжевый акцент
- L3 (Полезное): стандартные, зелёный акцент
- L4 (Контекст): компактные, серый акцент

**Дополнительные детали:**
- В правом верхнем углу: live-дата обновления + кнопка "← Канбан" (ссылка на /map.html)
- Поиск по карточкам (простой JS filter по тексту) в шапке
- Теги с кол-вом: "🔴 4 критических риска" — кликабельные фильтры
- Мобильная версия: карточки складываются в вертикальный аккордеон

**Соединения между уровнями:**
Вместо простых линий — SVG-стрелки с небольшой анимацией движения вниз (как flow-diagram). Линии должны исходить из нижней точки родительской карточки и приходить в верхнюю точку дочерней.

### Технические ограничения
- Только HTML + CSS + vanilla JS (никаких фреймворков, никакого Node.js)
- CDN допустим для иконок и шрифтов
- Файл должен работать как standalone (открыть локально или через nginx)
- Не использовать внешние CSS-фреймворки (Bootstrap, Tailwind) — только custom CSS

### Что вернуть
Верни полный HTML-файл готовый к деплою. Без объяснений, только код.
Все тексты контента бери из приложенного файла без изменений.
